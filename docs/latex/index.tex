egs\+\_\+brachy is an egs++ application for rapid brachytherapy calculations for both photon and electron sources. The current documentation serves as a Technical Reference Manual to complement the egs\+\_\+brachy user manual (\href{https://clrp-code.github.io/egs_brachy/pdf/egs_brachy_user_manual.pdf}{\texttt{https\+://clrp-\/code.\+github.\+io/egs\+\_\+brachy/pdf/egs\+\_\+brachy\+\_\+user\+\_\+manual.\+pdf}}) and our initial egs\+\_\+brachy paper (\href{https://clrp-code.github.io/egs_brachy/pdf/egs_brachy_paper2016.pdf}{\texttt{https\+://clrp-\/code.\+github.\+io/egs\+\_\+brachy/pdf/egs\+\_\+brachy\+\_\+paper2016.\+pdf}})\+: MJP Chamberland, REP Taylor, DWO Rogers, and RM Thomson, egs\+\_\+brachy\+: a versatile and fast Monte Carlo code for brachytherapy, Phys. Med. Biol. 61, 8214-\/8231 (2016). Please cite this paper when egs\+\_\+brachy is used in publications.

When using source or eye-\/plaque models from the distributed geometry library, please also cite the relevant CLRP database papers (see \doxysectlink{index_clrprefs}{CLRP database references}{3} below).

egs\+\_\+brachy was written by \href{http://randlet.com}{\texttt{Randle Taylor}} in collaboration with Marc Chamberland, Dave Rogers, and Rowan Thomson of the \href{http://www.physics.carleton.ca/clrp/}{\texttt{Carleton Laboratory for Radiotherapy Physics}}.

\hypertarget{index_usage}{}\doxysection{\texorpdfstring{Usage}{Usage}}\label{index_usage}
\hypertarget{index_runcontrol}{}\doxysubsection{\texorpdfstring{Run Control}{Run Control}}\label{index_runcontrol}
egs\+\_\+brachy uses the same run control block as other egs++ user codes with the addition of one extra input {\ttfamily egsdat file format} which you can use to tell egs\+\_\+brachy to output its egsdat file in gzip format. Using gzip format can result in significantly smaller egsdat file sizes for simulations with a large number of regions defined.

\begin{DoxyVerb}:start run control:
    ncase = 10000
    nbatch=1
    geometry error limit = 10
    calculation = first # first or restart or combine or analyze
    egsdat file format = gzip # gzip or text
:stop run control:
\end{DoxyVerb}


Detailed descriptions of the standard input options are available in \href{https://nrc-cnrc.github.io/EGSnrc/doc/pirs898/common.html}{\texttt{PIRS-\/898}}.\hypertarget{index_runmodes}{}\doxysubsection{\texorpdfstring{Run Modes}{Run Modes}}\label{index_runmodes}
There are currently three different run modes available in egs\+\_\+brachy\+:


\begin{DoxyEnumerate}
\item \textquotesingle{}normal\textquotesingle{} (default)\+: a regular simulation. For an example see \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/eb_tests/seeds_in_xyz/seeds_in_xyz.egsinp}{\texttt{eb\+\_\+tests/seeds\+\_\+in\+\_\+xyz/seeds\+\_\+in\+\_\+xyz.\+egsinp}}.
\item \textquotesingle{}superposition\textquotesingle{} \+: This type of simulation requires the used of an EGS\+\_\+\+ASwitched\+Envelope as the simulation geometry. At the start of every history only a single inscribed geometry will be activated. This allows you to explore the effects of intersource attenuation by performing TG-\/43 dose superposition type calculations. For an example see \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/eb_tests/tg43mode/tg43mode.egsinp}{\texttt{eb\+\_\+tests/tg43mode/tg43mode.\+egsinp}}.
\item \textquotesingle{}volume correction only\textquotesingle{}\+: Just run the volume correction routines, print the results and then quit. No actual dose calculations will be done, so the number of histories specified in the run control input block (ncase) is ignored. For an example see \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/eb_tests/volume_correction/vc.egsinp}{\texttt{eb\+\_\+tests/volume\+\_\+correction/vc.\+egsinp}}.
\end{DoxyEnumerate}

The run mode is set using a \textquotesingle{}run mode\textquotesingle{} input block\+:

\begin{DoxyVerb}:start run mode:

    run mode = normal # 'normal', 'superposition', or 'volume correction only'

:stop run mode:
\end{DoxyVerb}


In simulations with more than a single source, egs\+\_\+brachy uses the first source as a particle generator. That is, all particles are initiated at the location of the first source and transported until they escape the source geometry. From there, the particle is moved to the location of the next source before transport continues (more accurately, a new particle is added to the top of the stack at the location of the chosen source and the original particle is killed).

This behaviour can be disabled by setting {\ttfamily single generator = no} in the run mode block (see \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/eb_tests/single_generator/single_generator.egsinp}{\texttt{eb\+\_\+tests/single\+\_\+generator/single\+\_\+generator.\+egsinp}}).

\begin{DoxyVerb}:start run mode:
    run mode = normal
    single generator = no
:stop run mode:
\end{DoxyVerb}


Disabling the single generator (see \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/eb_tests/single_generator/multiple_generator.egsinp}{\texttt{eb\+\_\+tests/single\+\_\+generator/multiple\+\_\+generator.\+egsinp}}) mode may be slightly faster for some simulations (there is an extra call to howfar for every particle escaping the source geometry in single generator mode). Note that in some situations (particle recycling for example) single generator will always be enabled by egs\+\_\+brachy.\hypertarget{index_geomspec}{}\doxysubsection{\texorpdfstring{Geometry Specification}{Geometry Specification}}\label{index_geomspec}
In general, arbitrary egs++ geometries can be included in a simulation although, for the sake of efficiency, it is expected that EGS\+\_\+\+AEnvelope geometries will be used for most simulations.

There are three egs\+\_\+brachy specific input keys that are required for the \textquotesingle{}geometry definition\textquotesingle{} input block (in addition to the standard egs++ \textquotesingle{}simulation geometry\textquotesingle{} key)\+:


\begin{DoxyEnumerate}
\item \textquotesingle{}source geometries\textquotesingle{} \+: this key specifies which geometries define the actual brachytherapy source object. This may be a single geometry name (e.\+g. when using the {\ttfamily egs\+\_\+glib} shim to load an externally defined geometry) or a list of all the sub-\/geometries used to compose a single source geometry when definining geometries inline.
\item \textquotesingle{}phantom geometries\textquotesingle{}\+: this tells egs\+\_\+brachy which geometries to score dose in (1 or more phantom geometries are required). Currently 3 geometry types are allowed to be used as phantoms\+:
\begin{DoxyEnumerate}
\item EGS\+\_\+\+XYZGeometry (library egs\+\_\+ndgeometry, see \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/eb_tests/seeds_in_xyz/seeds_in_xyz.egsinp}{\texttt{eb\+\_\+tests/seeds\+\_\+in\+\_\+xyz/seeds\+\_\+in\+\_\+xyz.\+egsinp}})
\item EGS\+\_\+\+RZGeometry (library egs\+\_\+rz, see \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/eb_tests/stepped_source/stepped.egsinp}{\texttt{eb\+\_\+tests/stepped\+\_\+source/stepped.\+egsinp}})
\item EGS\+\_\+c\+Spheres (library egs\+\_\+spheres, see \href{../eb_tests/scatter/scatter.egsinp}{\texttt{eb\+\_\+tests/scatter/scatter.\+egsinp}})
\end{DoxyEnumerate}
\item \textquotesingle{}source envelope geometry\textquotesingle{}\+: this input is only required for superposition run mode and must name the EGS\+\_\+\+ASwitched\+Envelope geometry that contains the sources.
\end{DoxyEnumerate}

Transformed EGS\+\_\+\+XYZGeometry phantoms (i.\+e., EGS\+\_\+\+XYZGeometryT) are allowed, however, the number of voxels in each direction and their bounds are not reported in any of the output due to current limitations in the egs++ geometry library. A warning to that effect is output when a transformed XYZ geometry is used as a phantom in a simulation. Other phantom geometry types could be added in future provided they implement the get\+Bound, get\+NReg\+Dir, and get\+Mass methods of the EGS\+\_\+\+Base\+Geometry class.

An (abbreviated) example geometry specification might look like\+: \begin{DoxyVerb}:start geometry definition:

  # all geometries that are a part of the source
  source geometries = planes, end_cap_1, seed_middle, end_cap_2, seed

  phantom geometries = phantom

  simulation geometry = phantom_w_seeds

  # only required for superposition run mode
  source envelope geometry = phantom_w_seeds

  :start geometry:

      name = phantom
      library = egs_ndgeometry
      type    = EGS_XYZGeometry

      # rest of geom definition

  :stop geometry:

  :start geometry:
      library = egs_planes
      name = planes
      # ...
  :stop geometry:

  :start geometry:
        name = end_cap_1
        #...
  :stop geometry:

  :start geometry:
        name = seed_middle
        #...
  :stop geometry:

  :start geometry:
        name = end_cap_2
        #...
  :stop geometry:

  :start geometry:

      library = egs_cdgeometry
      name = seed
      base geometry = planes
      set geometry = 0 end_cap_1
      set geometry = 1 seed_middle
      set geometry = 2 end_cap_2

  :stop geometry:

  :start geometry:

      library = egs_autoenvelope
      name = phantom_w_seeds
      base geometry = phantom

      :start inscribed geometry:

          inscribed geometry name = seed

          # rest of auto envelope inscribed geom definition

      :stop inscribed geometry:

  :stop geometry:

:stop geometry definition:
\end{DoxyVerb}


Detailed descriptions of the egs++ geometry classes are available in \href{https://nrc-cnrc.github.io/EGSnrc/doc/pirs898/common.html}{\texttt{PIRS-\/898}}, including instructions on how to assign media. Note that vacuum can be assigned to a region by using the egs++ built-\/in name {\ttfamily vacuum}.\hypertarget{index_ctdata}{}\doxysubsubsection{\texorpdfstring{Using CT data to create phantoms}{Using CT data to create phantoms}}\label{index_ctdata}
Using the {\ttfamily egs\+\_\+glib} geometry library you can construct an {\ttfamily EGS\+\_\+\+XYZGeometry} using a {\ttfamily .egsphant} file like so\+:

\begin{DoxyVerb}:start geometry:
    library = egs_glib
    type = egsphant
    name = my_egsphant_geom
    egsphant file = /path/to/some/egsphant/file
    density file = /path/to/density/file
:stop geometry:
\end{DoxyVerb}


The egsphant file may either be a typical .egsphant text file or gzipped .egsphant.\+gz file, and the latter compressed format is often advantageous in terms of memory) The density file indicates to egs\+\_\+brachy the nominal density of each medium in the egsphant (voxel-\/by-\/voxel densities are defined in the egsphant file), and currently egs\+\_\+brachy can read these data from a (a) material.\+dat file, (b) pegs4 data file, or (c) a simple text file of the format\+:

\begin{DoxyVerb}MEDIUM=WATER
RHO=1.000
MEDIUM=AIR
RHO=1.2E-3
\end{DoxyVerb}


{\itshape Note the egs\+\_\+glib geometry library in the main EGSnrc repository does not currently contain support for egsphant files. The egs\+\_\+glib contained in the egs\+\_\+brachy git branch is customized to include support for egsphant files.}

It is also possible to use the egs\+\_\+ndgeometry library to contstruct an {\ttfamily EGS\+\_\+\+XYZGeometry} using a {\ttfamily .egsphant} file like so\+:

\begin{DoxyVerb}:start geometry:

    library = egs_ndgeometry
    type =  EGS_XYZGeometry
    name = my_egsphant_geom
    egsphant file = /path/to/some/egsphant/file
    ct ramp = /path/to/ramp/file.ramp

:stop geometry:
\end{DoxyVerb}


See the {\ttfamily egs\+\_\+ndgeometry} documentation for details about the format of the ct ramp file. Note, however, that the egs\+\_\+ndgeometry approach assumes that there is no overlap in voxel mass densities for different media in the egsphant, i.\+e., each medium has a distinct and non-\/overlapping range of mass densities. This is often {\itshape not} the case for brachytherapy phantoms, and so using the {\ttfamily egs\+\_\+glib} geometry with {\ttfamily .egsphant} files (gzip format) is recommended.\hypertarget{index_geomlib}{}\doxysubsubsection{\texorpdfstring{The geometry library}{The geometry library}}\label{index_geomlib}

The \textquotesingle{}scoring options\textquotesingle{} input block currently has the following keys\+:


\begin{DoxyItemize}
\item \textquotesingle{}muen file\textquotesingle{}\+: path to the file containing the muen data required for the simulation. See the Muen\+Data\+Parser class for more information about the format of this file.
\item \textquotesingle{}muen for media\textquotesingle{}\+: (Optional) a list of the materials that muen data is required for. Dose will be set to 0 for any phantom media that don\textquotesingle{}t have muen data available. (Can be used instead of or in conjunction with {\ttfamily \doxylink{namespacemuen}{muen} for medium}).
\item \textquotesingle{}muen for medium\textquotesingle{}\+: (Optional) one or more input blocks which specify the muen dataset to be used for a given transport medium. (Can be used instead of or in conjunction with {\ttfamily \doxylink{namespacemuen}{muen} for media}) This allows you to, for example, transport particles in tissue but score in water with notation Dw,m according to the conventions of TG-\/186.
\end{DoxyItemize}

        scoring medium = WATER
    :stop muen for medium:
\end{DoxyVerb}



\begin{DoxyItemize}
\item \textquotesingle{}score tracklength dose\textquotesingle{} \+: (Optional) Controls whether dose using a tracklength estimator is scored in the phantom geometries. The choices are \textquotesingle{}yes\textquotesingle{} (default) or \textquotesingle{}no\textquotesingle{}.
\item \textquotesingle{}score energy deposition\textquotesingle{} \+: (Optional) Controls whether dose from particle interactions is scored in the phantom geometries. The choices are \textquotesingle{}yes\textquotesingle{} or \textquotesingle{}no\textquotesingle{} (default). The choices are \textquotesingle{}yes\textquotesingle{} or \textquotesingle{}no\textquotesingle{} (default). The formalism is intendend for single source characterization. This option should only be used for single source simulations without recycling and with particles initialized within the source (i.\+e., not from a phase-\/space source). See \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/eb_tests/scatter/scatter.egsinp}{\texttt{eb\+\_\+tests/scatter/scatter.\+egsinp}} for an example.
\item \textquotesingle{}spectrum scoring\textquotesingle{}\+: (Optional) zero or more input blocks specifying spectra to score (described below). \item \textquotesingle{}phsp scoring\textquotesingle{}\+: (Optional) zero or one input block specifying whether to score a phase space on the external surface of the source (e.\+g., on the capsule of brachytherapy seeds). See \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/eb_tests/phsp_scoring/phsp_score.egsinp}{\texttt{eb\+\_\+tests/phsp\+\_\+scoring/phsp\+\_\+score.\+egsinp}} for an example.
\item \textquotesingle{}dose file format\textquotesingle{}\+: (Optional) Controls whether 3dddose files are written as text or gzipped text files. Options are \textquotesingle{}text\textquotesingle{} (default) and \textquotesingle{}gzip\textquotesingle{}.
\item \textquotesingle{}output egsphant files\textquotesingle{}\+: (Optional) Controls whether egsphant files are created for scoring phantoms. Options are \textquotesingle{}no\textquotesingle{} (default) and \textquotesingle{}yes\textquotesingle{}.
\item \textquotesingle{}egsphant file format\textquotesingle{}\+: (Optional) Controls whether egsphant files are written as text or gzipped text files. Options are \textquotesingle{}text\textquotesingle{} (default) and \textquotesingle{}gzip\textquotesingle{}.
\item \textquotesingle{}output voxel info files\textquotesingle{}\+: (Optional) Controls whether voxel info files are created for scoring phantoms. \item \textquotesingle{}voxel info file format\textquotesingle{}\+: (Optional) Controls whether voxel info files are written in text (default) or gzip format

\hypertarget{index_srccoord}{}\doxysubsubsection{\texorpdfstring{Source coordinate transform}{Source coordinate transform}}\label{index_srccoord}
The optional {\ttfamily source coordinate transform} block maps all source locations from the coordinate system used by the {\ttfamily transformations} block into the simulation coordinate system. This is useful when source placements are defined in a local frame (for example, seed positions relative to an eye plaque) and a separate transform places the plaque in the simulation.

The block contains exactly one standard egs++ {\ttfamily \+:start transformation\+:} input. At initialization, egs\+\_\+brachy composes this transform with each entry from {\ttfamily transformations} so that the world source location is $ T_{\mathrm{world},i} = T_{\mathrm{coord}} \cdot T_{\mathrm{seed},i} $. This matches the geometry workflow where {\ttfamily egs\+\_\+gtransformed} wraps an {\ttfamily egs\+\_\+autoenvelope} containing transformed seeds.

An example for an eye plaque simulation is distributed with egs\+\_\+brachy as \href{lib/geometry/eye_plaques/sample\%20eye\%20plaque\%20input\%20template\%20\%20files/COMS16mm-I125-6711-hetero-splitTransforms.egsinp}{\texttt{COMS16mm-\/\+I125-\/6711-\/hetero-\/split\+Transforms.\+egsinp}}. For setups in which the autoenvelope base geometry is the untransformed plaque, the pose file can instead be \href{lib/geometry/transformations/eye_plaques/COMS16mm_phantom_pose}{\texttt{COMS16mm\+\_\+phantom\+\_\+pose}}.

\begin{DoxyVerb}:start source definition:

  :start source:
      library = egs_isotropic_source
      name = 6711
      ...
  :stop source:

  :start transformations:
      include file = lib/geometry/transformations/eye_plaques/COMS16mm
  :stop transformations:

  :start source coordinate transform:
      :start transformation:
          include file = lib/geometry/transformations/eye_plaques/COMS16mm_example_pose
      :stop transformation:
  :stop source coordinate transform:

  simulation source = 6711

:stop source definition:
\end{DoxyVerb}


The existing {\ttfamily transformations} block is unchanged and remains required for backward compatibility. Do not use {\ttfamily egs\+\_\+transformed\+\_\+source} for this workflow; egs\+\_\+brachy applies source placements from the {\ttfamily transformations} block after sampling the base source.

The intended workflow uses the same two-\/level transform structure in both the geometry definition and the source definition. Seed positions relative to an applicator (for example, an eye plaque) are listed once, typically via the same {\ttfamily include file} in both places. In the geometry block, an {\ttfamily egs\+\_\+autoenvelope} inscribes each seed in the applicator using those local transforms. An outer {\ttfamily egs\+\_\+gtransformed} wrapper around the autoenvelope assembly then applies the applicator pose in the simulation frame (see \href{lib/geometry/eye_plaques/sample\%20eye\%20plaque\%20input\%20template\%20\%20files/COMS16mm-I125-6711-hetero-splitTransforms.egsinp}{\texttt{COMS16mm-\/\+I125-\/6711-\/hetero-\/split\+Transforms.\+egsinp}} for a worked example). In the source definition, the {\ttfamily transformations} block lists the same local seed placements, while {\ttfamily source coordinate transform} carries the same affine transform as the outer {\ttfamily egs\+\_\+gtransformed} wrapper. The per-\/seed geometry inscriptions and the source {\ttfamily transformations} entries must agree; the outer {\ttfamily egs\+\_\+gtransformed} and {\ttfamily source coordinate transform} blocks must agree with each other.

A regression test in {\ttfamily eb\+\_\+tests/source\+\_\+coordinate\+\_\+transform/} verifies that composed transforms reproduce the {\ttfamily seeds\+\_\+in\+\_\+xyz} gold-\/standard dose. egs\+\_\+brachy has the ability to reuse particles which escape from the source geometry an arbitrary number of times.     :stop particle recycling:
:stop variance reduction:
\end{DoxyVerb}

\hypertarget{index_voxvc}{}\doxysubsection{\texorpdfstring{Voxel volume correction details}{Voxel volume correction details}}\label{index_voxvc}
egs\+\_\+brachy has three methods available for doing voxel volume correction calculations in rectilinear, cylindrical and spherical geometries. There is a \textquotesingle{}fast\textquotesingle{} method that uses the same technique described in the egs\+\_\+autoenvelope documentation and a more general purpose routine which can be used for larger volumes with multiple overlapping phantom geometries. In addition to those two methods, corrected voxel volumes can be precomputed (either manually or by egs\+\_\+brachy) and read from a file. Finally, there is a fourth method, {\ttfamily phantom region volumes} that can be used to specify manually calcualted voxel volumes in the input file for other geometry types (e.\+g. conical geometries).

See \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/eb_tests/volume_correction/vc.egsinp}{\texttt{eb\+\_\+tests/volume\+\_\+correction/vc.\+egsinp}} for examples of the fast \& general volume correction methods.\hypertarget{index_coverthresh}{}\doxysubsubsection{\texorpdfstring{Total coverage threshold}{Total coverage threshold}}\label{index_coverthresh}
Volume correction uses Monte Carlo sampling to estimate how much of each phantom voxel remains available for dose scoring after subtracting overlapping geometry (sources, applicators, etc.). For voxels that are almost entirely occupied, the tiny uncorrected volume that remains can produce dose values with very large statistical uncertainties.

The optional input {\ttfamily total coverage threshold \%} controls when a voxel is treated as fully covered and its corrected volume set to zero (so no dose is scored there). If the estimated uncorrected fraction of the nominal voxel volume is at most {\ttfamily (100 -\/ threshold)} percent, the voxel is zeroed. With the default {\ttfamily 99.\+9}, voxels with 0.\+1\% or less of their nominal volume remaining are zeroed; with {\ttfamily 98}, voxels with 2\% or less remaining are zeroed. A lower threshold is sometimes used for complex applicators such as eye plaques, where slightly more aggressive zeroing avoids unreliable dose in thin voxels at geometry boundaries.

This input may appear in either {\ttfamily \+:start source volume correction\+:} or {\ttfamily \+:start extra volume correction\+:} blocks. Values may be given as a percentage ({\ttfamily 99.\+9}, {\ttfamily 98}) or as a fraction between 0 and 1. If omitted, egs\+\_\+brachy uses 99.\+9\%.

See \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/eb_tests/volume_correction/vc.egsinp}{\texttt{vc.\+egsinp}} for a test input with a non-\/default threshold on an {\ttfamily extra volume correction} block, and the eye-\/plaque template \href{https://github.com/clrp-code/egs_brachy/blob/main/egs_brachy/lib/geometry/eye_plaques/sample\%20eye\%20plaque\%20input\%20template\%20\%20files/BEBIG16mm-hetero-I125-S06_3rdSeedActive-.egsinp}{\texttt{BEBIG16mm-\/hetero-\/\+I125-\/\+S06\+\_\+3rd\+Seed\+Active-\/.\+egsinp}} for a clinical geometry example using {\ttfamily 98}.

\begin{DoxyVerb}:start volume correction:

    :start extra volume correction:

        correction type = correct
        density of random points (cm^-3) = 1E8
        total coverage threshold % = 98   # zero voxels with <= 2% uncorrected volume

        :start shape:
            type = box
            box size = 4
        :stop shape:

    :stop extra volume correction:

:stop volume correction:
\end{DoxyVerb}
\hypertarget{index_sourcevc}{}\doxysubsubsection{\texorpdfstring{Fast voxel volume corrections for sources}{Fast voxel volume corrections for sources}}\label{index_sourcevc}
The input block for this type of volume correction looks like\+:

\begin{DoxyVerb}:start volume correction:

    :start source volume correction:

        correction type = correct # optional: none(default), correct, zero dose
        density of random points (cm^-3) = 1E7 # optional random point sampling density defaults to 1E8

        total coverage threshold % = 99.9 # optional level of voxel coverage before it is considered totally covered
                                          # if a voxel is more than 99.9% (default) covered then it's volume is set to 0

        # shape which encompasses source
        :start shape:
            type = cylinder
            radius = 0.04
            height = 0.45
        :stop shape:

        # optional rng definition
        :start rng definition:
            type = sobol
            initial seed = 1234
        :stop rng definition:

        # -or- #
        :start rng definition:
            type = ranmar
            initial seeds = two integers
        :stop rng definition:

    :stop source volume correction:

:stop volume correction:
\end{DoxyVerb}
\hypertarget{index_genvc}{}\doxysubsubsection{\texorpdfstring{General purpose volume corrections}{General purpose volume corrections}}\label{index_genvc}
The general purpose algorithm is similar, except any geometry region within the bounding shape will have its region volume corrected Currently the shape must be cylinder, sphere, or box type.

The input block for the general purpose volume corrections is shown below\+:

\begin{DoxyVerb}:start volume correction:

    :start extra volume correction:

        correction type = correct # correct, none, zero dose
        density of random points (cm^-3) = 1E5
        total coverage threshold % = 98   # optional; see #coverthresh (default 99.9)

        # correct volume of region for all geometries within
        # x,y,z = +/- 2cm around origin
        :start shape:

            type = box
            box size = 4

            :start transformation:
                translation = 0 0 0
            :stop transformation:

        :stop shape:

    :stop extra volume correction:

:stop volume correction:
\end{DoxyVerb}
\hypertarget{index_vcfile}{}\doxysubsubsection{\texorpdfstring{Volume calculations from file}{Volume calculations from file}}\label{index_vcfile}
You can also use precomputed volume corrections by using a \textquotesingle{}volume correction from file\textquotesingle{} input.

The input block specifies one or more \textquotesingle{}phantom file\textquotesingle{} inputs with two strings. The first string is the name of the phantom to set the volumes for and the second string is the file to read the volumes from. egs\+\_\+brachy will automatically determine whether the file is in text or gzip mode.

\begin{DoxyVerb}:start volume correction:

    :start volume correction from file:

        phantom file = phantom1 your_precomputed_phantom1_volumes.volcor
        phantom file = phantom2 your_precomputed_phantom2_volumes.volcor

    :stop volume correction from file:

:stop volume correction:
\end{DoxyVerb}


To create your own volcor text files, create a text file in the format\+:

\begin{DoxyVerb}NRECORDS
REG_NUM_1 REG_1_VOLUME
REG_NUM_2 REG_2_VOLUME
...
REG_NUM_3 REG_3_VOLUME
\end{DoxyVerb}


for example the following file\+:

\begin{DoxyVerb}3
5 0.5
13 1.0
1000 2.5
\end{DoxyVerb}


would set regions 5, 13 and 1000 to volumes 0.\+5 cm\texorpdfstring{$^\wedge$}{\string^}3, 1 cm\texorpdfstring{$^\wedge$}{\string^}3, 2.\+5 cm\texorpdfstring{$^\wedge$}{\string^}3 of whichever phantom it was assigned to.\hypertarget{index_rngvc}{}\doxysubsubsection{\texorpdfstring{Random Number Generator for volume corrections}{Random Number Generator for volume corrections}}\label{index_rngvc}
If a {\ttfamily box} shape is used as the volume correction shape, volume corrections will use a Sobol Quasi Random Number Generator by default. For all other shapes the regular Ranmar RNG will be used (the Sobol generator only works for Cartesian coordinate systems). The RNG can be overridden as show in the example above.\hypertarget{index_vcuser}{}\doxysubsubsection{\texorpdfstring{Manually specifying voxel volumes}{Manually specifying voxel volumes}}\label{index_vcuser}
For calculating dose in phantoms which are not rectilinear, cylindrical, or spherical you may use {\ttfamily phantom region volumes} blocks to specify the the volume of one or more regions in a phantom (note, it will also work in e.\+g. rectilinear phantoms, it\textquotesingle{}s just not generally needed). The format of these blocks is\+:

\begin{DoxyVerb}:start volume correction:

    :start phantom region volumes:
        phantom name = <phantom_name>
        region numbers = r_1 r_2 ... r_i ... r_N
        region volumes = V_1 V_2 ... V_i ... V_N
    :stop phantom region volumes:

:stop volume correction:
\end{DoxyVerb}


for example to specify volumes of 0.\+5cm\texorpdfstring{$^\wedge$}{\string^}3 and 1cm\texorpdfstring{$^\wedge$}{\string^}3 for regions 1 and 3 respectively in a phantom called {\ttfamily my\+\_\+phantom} you would do\+:

\begin{DoxyVerb}:start volume correction:

    :start phantom region volumes:
        phantom name = my_phantom
        region numbers = 1 3 
        region volumes = 0.5 1
    :stop phantom region volumes:

:stop volume correction:
\end{DoxyVerb}


if you try to calculate dose in a geometry type that doesn\textquotesingle{}t support automatic volume calculations without including a {\ttfamily phantom region volumes} block, egs\+\_\+brachy will print a warning and terminate.\hypertarget{index_outputfiles}{}\doxysubsection{\texorpdfstring{Output files}{Output files}}\label{index_outputfiles}
\hypertarget{index_dosefiles}{}\doxysubsubsection{\texorpdfstring{3ddose files}{3ddose files}}\label{index_dosefiles}
A 3ddose file will be output for every phantom geometry named in the {\ttfamily geometry definition -\/\texorpdfstring{$>$}{>} phantom geometries} input item. The filename format of these 3ddose files is {\ttfamily \{input\+\_\+file\}\+\_\+\{phantom\+\_\+name\}.3ddose} where {\ttfamily \{input\+\_\+file\}} is replaced with the name of the simulations input file and {\ttfamily \{phantom\+\_\+name\}} is replaced with the name of the phantom.

If {\ttfamily score energy deposition = yes} is set, a second 3ddose file with dose from interaction scoring will be output to {\ttfamily \{input\+\_\+file\}\+\_\+\{phantom\+\_\+name\}.edep.\+3ddose} as well.

The 3ddose file format is described in section 6.\+2 of the \href{https://clrp-code.github.io/egs_brachy/pdf/egs_brachy_user_manual.pdf}{\texttt{egs\+\_\+brachy manual}}. By default, the doses are reported in units of Gray per history (Gy/hist). If a dose scaling factor is provided within the egsinp file to convert the results to absolute dose as per section 7.\+4.\+2 of the egs\+\_\+brachy manual, then the units of the doses will simply be Gy.

If {\ttfamily score scatter dose} is enabled, egs\+\_\+brachy will score primary, single scattered and multiple scattered dose (normalized to total radiant energy) and output them to 3ddose files with the format {\ttfamily \{input\+\_\+file\}\+\_\+\{phantom\+\_\+name\}.\{pr,ss,ms,to\}.3ddose} The units of the doses are Gray per radiant energy (Gy/R) or simply inverse grams (g\texorpdfstring{$^\wedge$}{\string^}-\/1).

Note that for voxels with volume corrections, the uncertainty in the 3ddose file represents the total uncertainty of the calculation which is the error on the dose calculation added in quadrature to the error on the volume correction calculation.\hypertarget{index_egsphant}{}\doxysubsubsection{\texorpdfstring{egsphant file}{egsphant file}}\label{index_egsphant}

\end{DoxyVerb}
